## 📖 מבוא

### תיאור המערכת

**RideFlow** היא מערכת ייעודית לניהול ורישום לתחבורה ציבורית / פרטית (היסעים).  
המערכת מאפשרת ניהול יעיל של נסיעות, נוסעים, נהגים, כלי רכב ותחנות, תוך חוויית משתמש נוחה ואינטואיטיבית.

### נתונים הנשמרים במערכת

| ישות | נתונים עיקריים |
|---|---|
| **נוסע (Passenger)** | שם מלא, אימייל, טלפון, מגזר |
| **נהג (Driver)** | שם מלא, מזהה, סוג רישיון |
| **כלי רכב (Vehicle)** | מספר לוחית רישוי, סוג רכב, קיבולת |
| **מסלול (Route)** | שם המסלול, מזהה |
| **תחנה (Stop)** | שם תחנה, מזהה |
| **נסיעה (Trip)** | תאריך, שעת יציאה, מושבים פנויים |
| **רישום (Registration)** | מזהה נוסע, מזהה נסיעה, תחנת עלייה, תחנת ירידה, סטטוס |
| **כולל (Includes)** | קשר מסלול-תחנה |

### פונקציונאליות עיקרית

- **רישום לנסיעות** — הנוסע יכול לחפש נסיעה מתאימה ולהירשם אליה בקלות
- **ניהול הרשמות** — צפייה, ביטול ועדכון הרשמות קיימות
- **דשבורד נתונים** — ממשק ויזואלי המציג סטטיסטיקות על הנסיעות וההרשמות
- **ניהול מסלולים ותחנות** — הגדרת קווים ותחנות הנכללות בכל קו
- **ניהול צי הרכבים** — שיוך נהגים ורכבים לנסיעות

## 🎨 החלטות עיצוב

### 1. עיצוב ממשק מבוסס AI (ShuttleGo)
השתמשנו בכלי AI Studio ליצירת ממשק מודרני ואינטואיטיבי.  
**נימוק:** עיצוב מקצועי ותחרותי שמוריד את זמן הפיתוח ומאפשר אב-טיפוס מהיר.

### 2. רשת טבלאות ממוקדת
פיצול המידע ל-8 טבלאות נפרדות (Driver, Vehicle, Route, Stop, Trip, Passenger, Registration, Includes).  
**נימוק:** מניעת כפילות נתונים (נורמליזציה), שמירה על עקביות ויכולת הרחבה עתידית.

### 3. שיוך תחנת עלייה וירידה לרישום
בטבלת `Registration` שמרנו גם את `boarding_stop_id` וגם את `dropoff_stop_id`.  
**נימוק:** כמה מסלולים כוללים עשרות תחנות — הנוסע צריך לציין את נקודות הכניסה והיציאה הספציפיות שלו.

### 4. שדה סטטוס ברישום
הוספנו שדה `status` לרישום (פעיל / מבוטל / ממתין).  
**נימוק:** מאפשר ביטול הרשמה מבלי למחוק רשומות מהיסטוריית הנתונים.

## 💾 שיטות הכנסת נתונים

נבחרו **3 שיטות** להכנסת נתונים למערכת:

---

### שיטה 1: כלי Mockaroo — יצירת נתוני דמי אוטומטית
יצירת נתונים אקראיים ריאליסטיים לטבלאות בפורמט CSV דרך האתר [mockaroo.com](https://mockaroo.com).

![שיטה 1 - Mockaroo](screemshots/WhatsApp%20Image%202026-04-15%20at%2014.42.13.jpeg)

---

### שיטה 2: ייבוא קובץ CSV דרך pgAdmin (Import/Export)
ייבוא קבצי ה-CSV שנוצרו ב-Mockaroo ישירות אל תוך הטבלאות במסד הנתונים PostgreSQL דרך ממשק pgAdmin.

![שיטה 2 - Import CSV](screemshots/WhatsApp%20Image%202026-04-15%20at%2014.43.37.jpeg)

---

### שיטה 3: Python Script — סקריפט Python לייצור נתונים
סקריפט Python ייעודי (`generate_data.py`) שמאפשר לבחור טבלה, כמות שורות, ומוסיף נתונים ישירות לבסיס הנתונים.

![שיטה 3 - Python Script](screemshots/WhatsApp%20Image%202026-04-15%20at%2014.51.15.jpeg)
